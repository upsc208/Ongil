# 🛠 온길 프로젝트 FE 리팩토링  

## 🚀 리팩토링 이유  
온길 프로젝트는 창의저인 아이디어를 바탕으로 개발되었지만, 성능 저하 및 코드 구조의 비효율성으로 인해 최적화가 필요했습니다.  
보다 빠르고 유지보수하기 쉬운 코드로 개선하여 **성능과 사용자 경험을 극대화**하는 것을 목표로 합니다.  

## 📌 주요 리팩토링 작업  
- ✅ **폴더 구조 변경** – 유지보수성을 높이기 위한 디렉토리 구조 정리  
- ✅ **상태관리 라이브러리 변경** – `Recoil` → `Zustand` 로 변경하여 성능 최적화  
- ✅ **불필요한 코드 제거** – 코드 가독성과 실행 속도 개선  
- ✅ **보안 강화** – 토큰 관리 방식 변경 (`LocalStorage` → `HttpOnly Cookie`)   
- ✅ **재사용 가능한 컴포넌트 추가** – 코드 중복을 줄이고 UI 일관성 유지
